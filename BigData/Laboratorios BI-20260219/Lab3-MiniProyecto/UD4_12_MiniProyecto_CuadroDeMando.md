# UD4 - Mini-Proyecto
## Diseño de Cuadro de Mando Profesional

Centro: IES Rafael Alberti
Módulo: Sistemas de Big Data
Curso: 2025-2026

---

# 1. Objetivo

Diseñar e implementar un cuadro de mando profesional a partir de un dataset real trabajado en unidades anteriores.

El objetivo no es generar gráficos aislados, sino construir un dashboard coherente que permita apoyar decisiones.

---

# 2. Alcance del proyecto

Este mini-proyecto debe:

- Utilizar un dataset real (mínimo 10.000 registros).
- Tener dimensión temporal obligatoria.
- Incluir al menos 5 columnas relevantes.
- Implementarse en Metabase o Superset.

No se permite utilizar datasets triviales o de prueba.

---

# 3. Parte 1 - Definición analítica (obligatoria antes de implementar)

Cada grupo debe entregar previamente:

1. Problema que se quiere analizar.
2. Público objetivo del dashboard.
3. Decisiones que debería permitir tomar.
4. Métricas que se van a utilizar.
5. Tipo de agregaciones previstas.

Sin esta parte aprobada no se puede continuar con la implementación.

---

# 4. Parte 2 - Implementación técnica

El dashboard final debe contener como mínimo:

1. Un KPI principal (tipo Stat o indicador numérico).
2. Una serie temporal.
3. Una comparación por categoría.
4. Un gráfico que permita segmentación o filtrado.
5. Organización clara y coherente.

No se evalúa estética avanzada.
Se evalúa coherencia analítica.

---

# 5. Parte 3 - Análisis e interpretación

Debe incluir:

- Interpretación de cada gráfico.
- Conclusiones generales.
- Limitaciones detectadas.
- Datos adicionales que mejorarían el análisis.

---

# 6. Presentación

Cada grupo realizará:

- Entrega en PDF.
- Exposición breve de 5-7 minutos.

---

# 7. Criterios de evaluación

Se evaluará:

- Capacidad analítica.
- Correcta definición de métricas.
- Coherencia del dashboard.
- Interpretación razonada.
- Claridad en la presentación.
